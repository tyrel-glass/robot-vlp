
\section{RNN}




A staged tuning approach that breaks down the hyperparameter search space into manageable subsets was used.

\section{Staged Hyperparameter Tuning Approach}

\subsection{Stage 1: Architecture Tuning}
In the first stage, the focus is on identifying the optimal network architecture. Key hyperparameters include:
\begin{itemize}
    \item \textbf{Sequence Length:} The number of timesteps used as input.
    \item \textbf{Number of Layers:} The depth of the RNN, tuned over a discrete range.
    \item \textbf{Units per Layer:} The number of neurons in each recurrent layer, often sampled on a logarithmic or exponential scale (e.g., \{2, 4, 8, 16, 32, 64, 128, 256\}).
    \item \textbf{Layer Type:} Choice between LSTM and GRU.
\end{itemize}

\subsection{Stage 2: Regularization Tuning}
Once a promising architecture is identified, the next stage is to optimize regularization strategies to prevent overfitting. This involves:
\begin{itemize}
    \item \textbf{Dropout and Recurrent Dropout:} Tuning dropout rates for both inputs and recurrent connections.
    \item \textbf{Normalization Techniques:} Evaluating the impact of batch normalization versus layer normalization.
\end{itemize}

\subsection{Stage 3: Optimization Tuning}
With the architecture and regularization settings fixed, the third stage focuses on the training dynamics:
\begin{itemize}
    \item \textbf{Learning Rate and Schedulers:} Tuning the initial learning rate and selecting between constant, exponential, or cosine decay schedules.
    \item \textbf{Optimizer Choice:} Evaluating different optimizers such as Adam, RMSprop, Nadam, and SGD.
\end{itemize}

\subsection{Stage 4: Additional Parameter Tuning}
Finally, additional hyperparameters, such as sequence length (if not fixed previously) and loss weighting for multi-output models, are fine-tuned. For instance, if one output's loss is naturally higher than the other, loss weights can be adjusted to balance their contributions during training.

\section{Implementation}
The staged tuning approach is implemented using Keras Tuner. Random search and grid search methods are applied selectively based on the size of the hyperparameter space. For example, when only a small number of sequence length values (e.g., \{10, 20, 30, 40, 50\}) need to be explored, grid search is employed to exhaustively evaluate each option.

\section{Conclusion}
The staged hyperparameter tuning strategy allows for a more systematic and interpretable optimization process. By decomposing the overall tuning problem into distinct stages, it becomes easier to isolate the impact of different hyperparameter groups on model performance. This approach not only improves efficiency but also provides deeper insights into the contributions of each model component.
